# NT2 Lezen B1 — Parafrase Lab

موقع PWA حديث لتدريب **Staatsexamen NT2 Programma I — Lezen B1** على إعادة الصياغة: كلمات مختلفة ومعنى واحد.

## ما الذي تغير؟

- السؤال والدليل والإجابة مخفية قبل المحاولة.
- **71 مثالًا** من 2020–2024 تستخدم الاختيارات الأصلية المستخرجة من ملفات الامتحان.
- **19 مثالًا** من نموذج 2025 تعمل كتقييم ذاتي واضح؛ لم تُخترع اختيارات غير موجودة في الملفات.
- عند الخطأ يظهر **Niet goed** ولا يُكشف الحل، وتبقى في السؤال نفسه.
- عند الصواب يظهر **Goed** ثم الدليل والرابط المعنوي والشرح العربي.
- رابط مباشر إلى صفحة الدليل في ملف PDF الأصلي لكل مثال.
- مراجعة قصيرة بعد كل خمسة أسئلة.
- حفظ المحاولات والأخطاء والمفضلة والكلمات غير المفهومة في IndexedDB.
- تصدير واستيراد نسخة احتياطية بصيغة JSON.
- بحث متعدد المجالات وفلاتر غير متعارضة.
- PWA قابلة للتثبيت وتعمل دون إنترنت بعد أول زيارة ناجحة للموقع المنشور.
- إشعار عند توفر تحديث جديد بدل بقاء نسخة قديمة في الذاكرة المؤقتة.

## التقنيات

- React 19
- TypeScript
- Vite
- React Router
- IndexedDB عبر `idb`
- Zod للتحقق من البيانات
- Workbox / vite-plugin-pwa
- CSS حديث: Layers، Container Queries، `color-mix()`، تصميم متجاوب ووضع داكن
- Vitest للاختبارات

الإصدارات الدقيقة المثبتة موجودة في `package-lock.json`.

## التشغيل المحلي

```bash
npm install
npm run dev
```

ثم افتح العنوان الذي يعرضه Vite.

## الفحص والبناء

```bash
npm run check
npm run build
npm run preview
```

المجلد الجاهز للنشر هو:

```text
dist/
```

## النشر

### Netlify

ارفع المشروع؛ ملف `netlify.toml` يحدد أمر البناء ومجلد النشر تلقائيًا.

### Vercel

ارفع المشروع؛ ملف `vercel.json` جاهز.

### GitHub Pages

ادفع المشروع إلى فرع `main`، ثم فعّل GitHub Pages باستخدام **GitHub Actions**. سير العمل موجود في:

```text
.github/workflows/deploy.yml
```

## ملاحظة مهمة

- التثبيت وService Worker يحتاجان HTTPS أو `localhost`.
- فتح `dist/index.html` مباشرة عبر `file://` لا يفعّل مزايا PWA كاملة.
- تقدم الطالب محفوظ محليًا، لذلك يُنصح بتصدير نسخة احتياطية قبل تغيير الجهاز أو مسح بيانات المتصفح.
