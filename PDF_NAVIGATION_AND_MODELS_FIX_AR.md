# إصلاح فتح PDF وقائمة النماذج التدريبية الحديثة

## سبب العودة إلى الصفحة الرئيسية

كان زر «فتح» يوجّه إلى ملف PDF خام داخل `public/sources`.
في نسخة PWA كان Workbox يسجّل NavigationRoute يعيد `index.html` لكل طلب تنقّل تقريبًا.
فتح ملف PDF في تبويب جديد يُرسل طلبًا من نوع navigation، ولذلك كان Service Worker يعيد صفحة React بدل الملف.
بعد ذلك يقرأ HashRouter مسارًا غير معروف ويحوّله إلى `/home`، فتظهر الصفحة الرئيسية.

## الإصلاح

1. أضيف مسار مستقل `#/pdf` يستخدم PDF.js ويعرض الملف في صفحة واسعة جديدة.
2. جميع أزرار «فتح» وروابط PDF ومفاتيح الإجابة أصبحت تستخدم صفحة القراءة المستقلة.
3. أضيف `navigateFallbackDenylist` لمسارات `/sources/` وملفات `.pdf` حتى لا يعيد Service Worker صفحة React بدل PDF.
4. نُقل Cache الخاص بملفات PDF قبل Cache الخاص بصفحات HTML، وتم استثناء PDF من قاعدة document.
5. أزيل قسم «نماذج تدريبية حديثة» من صفحة النماذج لأنه لم يكن تدريبًا تفاعليًا وكان يسبب التباسًا للطالب. بقيت الملفات في المصادر فقط لأغراض التوثيق.

## التحقق

- TypeScript: ناجح
- Vitest: 30/30
- Production build: ناجح

- مسار العارض الجديد: `/inburgeren/#/pdf?...`
